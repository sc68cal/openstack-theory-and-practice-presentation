OpenStack - Essex

Notes about Essex Architecture

  • Continued work on breaking compute service up into smaller parts
  • Original code base was monolithic - used to do everything
    • Identity
    • Object Storage
    • Networking
    • Block Storage
  • Original code base (nova) morphed into the Compute service
  • Code that handled everything besides compute was extracted from the nova codebase and became separate projects

OpenStack - Folsom

Notes about Folsom Architecture

  • Introduction of two new services:
    • Block Storage API (Cinder)
    • Network API (Quantum)
  • These services were extraced from the Compute service codebase
